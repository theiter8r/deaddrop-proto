How should the API structure be visualized?

Pick the approach that best fits what you want to explore

A

Collapsible Path Tree

Breaks endpoint paths into a nested filesystem-like tree. Each path segment is a node you can expand/collapse. Best for understanding hierarchy and navigating large APIs.

▼ /api
▼ /v1
▼ /users
● GET   /users
● POST /users
▼ /{id}
● GET   /users/{id}
● DEL   /users/{id}
▼ /auth
● POST /auth/login
● POST /auth/logout
B

Treemap (Area Chart)

A rectangular heatmap where each block is an API group, sized by the number of endpoints it contains. Great for spotting which service areas are largest or most complex.

/api/v1/users
12 endpoints
/auth
5 endpoints
/admin
4 endpoints
/health
/webhooks
C

Both — Toggle Between Views

Add a view switcher so users can flip between the tree hierarchy and the treemap. Tree for navigation depth, treemap for surface-area overview.
